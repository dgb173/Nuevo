Análisis Comparativo de Proxies v1
====================================

### Sección A: Tabla Comparativa de Servicios de Proxy

La siguiente tabla compara tres proveedores líderes para un escenario de ~10,000 peticiones al mes, considerando los requisitos específicos de este proyecto (necesidad de ejecutar `Selenium`/`Playwright`).

| Característica | Bright Data | ScraperAPI | Oxylabs |
| :--- | :--- | :--- | :--- |
| **Modelo de Precios** | Pago por Uso (Pay-As-You-Go) | Suscripción Mensual | Suscripción Mensual |
| **Coste Estimado (10k req/mes)** | **~$17 - $30** | ~$49 (Plan Hobby) | ~$49 (Plan Micro) |
| **Integración con `requests`** | Fácil (configuración de proxy) | Muy Fácil (SDK o endpoint de API) | Fácil (configuración de proxy) |
| **Integración con `Selenium`/`Playwright`** | **Directa y Soportada.** Permite configurar el proxy directamente en el navegador para un control total. | **Indirecta.** El modelo principal es enviar la URL a su API para que ellos la rendericen, no para que tú controles el navegador a través de su proxy. | **Directa y Soportada.** Similar a Bright Data, permite la configuración del proxy en el navegador. |
| **Ideal para este Proyecto** | **Sí.** Se alinea perfectamente con la arquitectura actual de controlar el navegador. | No. Requeriría una re-arquitectura significativa del scraper. | Sí. Es una alternativa viable a Bright Data. |

*Nota sobre costes: La estimación para Bright Data se basa en su calculadora de precios para proxies residenciales, asumiendo un uso de tráfico de bajo a moderado por petición. El coste real puede variar.* 

---

### Sección B: Justificación de la Elección

La elección de **Bright Data** se justifica porque su modelo de servicio es el único que permite una integración directa y sin fricciones con la arquitectura de scraping existente (basada en el control del navegador con `Selenium`/`Playwright`), además de ofrecer un modelo de pago por uso potencialmente más económico para un volumen de peticiones bajo o variable.
